# turbo-terminal-ios-claude-code

> Настройка рабочего окружения с Claude Code: полный набор для серверов (SSH + tmux), облегчённый для локальных машин (macOS)

**Версия:** 1.1.0

---

## Быстрый старт

Одна команда — клонирует репозиторий и запускает интерактивный инсталлятор:

```bash
git clone https://github.com/mad0ps/turbo-terminal-ios-claude-code.git /tmp/turbo-setup && bash /tmp/turbo-setup/setup.sh
```

> На сервере запускай **вне tmux**. Работает и на macOS — автоматически определит локальную машину и предложит только нужные компоненты.

Пробный запуск без изменений:
```bash
bash setup.sh --dry-run
```

---

## Что это?

Интерактивный Bash-инсталлятор, который автоматически определяет тип машины и предлагает подходящий набор инструментов.

### Два режима работы

**Удалённый сервер** (SSH) — полный набор для мобильной работы с iPhone:

| Компонент | Что делает |
|-----------|------------|
| **Claude Code** | Автоматическая установка через официальный скрипт |
| **tmux конфиг** | Prefix `Ctrl+A`, переключение окон стрелками, сплиты через `\|` и `-`, статус-бар наверху |
| **Алиасы** | `c` = claude, `cr` = resume, `cc` = continue, `q` = быстрый вопрос, `tl`/`tk`/`td` = tmux |
| **Навигация** | `p имя` — перейти в проект, `np имя` — создать папку + tmux-окно (с автокомплитом) |
| **Меню логина** | При SSH-подключении показывает сессии с именами окон |
| **Автосохранение** | tmux-resurrect + continuum — сессии сохраняются каждые 10 минут |
| **Скилы Claude Code** | Кастомные скилы, хуки и конфиг из [turbo-claude-code](https://github.com/mad0ps/turbo-claude-code) (опционально) |
| **Разрешения агентов** | Опциональная настройка `~/.claude/settings.json` (по умолч��нию выключено) |

**Локальная машина** (macOS) — только самое нужное:

| Компонент | Что делает |
|-----------|------------|
| **Claude Code** | Ус��ановка если не найден |
| **Алиасы** | `c`, `cr`, `cc`, `q` — быстрый доступ к Claude Code |
| **Навигация** | `p имя` — перейти в проект, `np имя` — соз��ать папку + cd (без tmux) |
| **Скилы Claude Code** | Кастомные скилы, хуки и конфиг из [turbo-claude-code](https://github.com/mad0ps/turbo-claude-code) (опционально) |
| **Разрешения агентов** | Опционально |

Определение режима — автоматическое по `$SSH_CONNECTION`. Каждый компонент ставится по выбору.

## Как работает инсталлятор

```
bash setup.sh
```

1. **Определяет систему** — ОС, пакетный менеджер (apt/dnf/yum/apk/brew), шелл (bash/zsh/fish), SSH или локальная
2. **Спрашивает что ставить** — на сервере: полный набор. На локальной машине: только алиасы и Claude Code
3. **Делает бэкап** — сохраняет текущие конфиги в `~/.terminal-setup-backup-<дата>/`
4. **Устана��ливает** — выбранные компоненты. Если пакет не установился — зависимые шаги отключаются автоматически
5. **Применяет** — выводит шпаргалку (разную для сервера и локальной машины)

### Безопасность

- Скрипт **идемпотентен** — повторный запуск безопасен, старый конфиг вырезается по маркерам и пишется заново
- Перед каждым изменением создаётся **бэкап** в `~/.terminal-setup-backup-<дата>/`
- `settings.json` при наличии **мержится** (добавляются только permissions), кастомные настройки (env, hooks, plugins) сохраняются. Требуется python3 для мержа; без python3 — перезаписывается с предупреждением
- При повторном запуске устаревшее разрешение `Task` автоматически мигрируется на `TodoWrite`

Откат к исходным настройкам:
```bash
cp ~/.terminal-setup-backup-*/* ~/
source ~/.bashrc
```

## Шпаргалка

### tmux (prefix = `Ctrl+A`)

| Действие | Команда |
|----------|---------|
| Список сессий | `Ctrl+A s` |
| Новое окно | `Ctrl+A n` |
| Следующее окно | `Ctrl+A ↓` |
| Предыдущее окно | `Ctrl+A ↑` |
| Вертикальный сплит | `Ctrl+A \|` |
| Горизонтальный сплит | `Ctrl+A -` |
| Переключение панелей | `Alt + стрелки` |
| Popup окно | `Ctrl+A g` |
| Закрыть панель | `Ctrl+A x` |
| Отключиться (detach) | `Ctrl+A d` |

### Алиасы

| Команда | Описание |
|---------|----------|
| `c` | Запустить Claude Code |
| `cr` | Выбрать и продолжить сессию Claude |
| `cc` | Продолжить последнюю сессию Claude |
| `q "вопрос"` | Быстрый во��рос Claude |
| `p` | Показать список проектов |
| `p имя` | Перейти в проект (с подсказкой если не найден) |
| `np имя` | Сервер: создать папку + tmux-окно. Локально: mkdir + cd |
| `tl` | Список tmux-сессий (только на сервере) |
| `tk имя` | Убить сессию (только на сервере) |

> **Совет:** Имена функций `p` и `np` можно переименовать при установке если конфликтуют с существующими алиасами.

### Меню логина (только на сервере)

При SSH-��одключении показывает сессии с именами окон:
```
  ● АКТИВНЫЕ СЕССИИ
  ───────────────────────────────
    ▸ 0 (2 окон) [money-game api-server]
    ▸ 1 (1 окон) [whoami]
  ───────────────────────────────
```
- **[a]** — подключиться к существующей сессии
- **[n]** — создать новую сессию
- **[r]** — переименовать сессию
- **[s]** — войти в shell без tmux
- Или просто введи имя сессии для быстрого входа

### Расширенные разрешения для агентов

При установке можно включить опцию `[y/N]` — разрешить Claude Code выполнять все действия без подтверждений. Создаёт или обновляет `~/.claude/settings.json` с разрешениями на все инструменты (Bash, Edit, Write, Read, Glob, Grep, WebFetch, WebSearch, NotebookEdit, TodoWrite, Skill). Рекомендуется только для доверенных серверов.

## Типичный рабочий процесс

**На сервере (iPhone + SSH):**
```
1. Открыл Termius → подключился по SSH → увидел меню сессий
2. Выбрал сессию или создал новую
3. cr → продолжил работу в Claude Code
4. Ctrl+A d → отключился, всё работает в фоне
5. Автосохранение каждые 10 минут
```

**На маке (локально):**
```
1. Открыл терминал
2. p имя → перешёл в проект
3. c → запустил Claude Code
```

## Советы для iPhone

- **Раскладка** — переключи на английский перед `Ctrl+A`, tmux не понимает русскую
- **Свайпы в Termius** — по экрану работают как стрелки
- **Landscape** — переверни телефон горизонтально когда нужно больше ширины
- **Шрифт** — уменьши в настройках Termius для большего контента
- **Сниппеты Termius** — сохрани частые команды, запуск в одно нажатие
