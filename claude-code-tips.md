## Как прокачать Claude Code — советы из практики

### Разрешения: чтобы не спрашивал каждый раз "можно?"

По умолчанию Claude Code перед каждым действием спрашивает подтверждение: "можно создать файл?", "можно выполнить команду?". Это безопасно, но дико раздражает когда работаешь активно.

Решение — файл настроек `~/.claude/settings.json`, где ты один раз говоришь "вот эти действия разрешаю всегда". После этого Claude работает без пауз — как помощник, а не как стажёр, который бегает за подписью на каждую бумажку.

**Как настроить:**
1. Открой Claude Code
2. Попроси его: "создай ~/.claude/settings.json с разрешениями для всех инструментов"
3. Или сделай сам — попроси Claude создать этот файл, он знает формат

После этого Claude перестанет спрашивать разрешение на каждый чих.

### Плагины: суперспособности из коробки

У Claude Code есть **плагины** — наборы готовых инструкций, которые учат его работать по-умному. Это как приложения для телефона — ставишь нужные и получаешь новые возможности.

Самый полезный плагин — **Superpowers** (от разработчика Jesse Vincent). Он добавляет 14 "скиллов":
- **Brainstorming** — прежде чем кодить, Claude обсуждает с тобой идею, предлагает варианты, и только после твоего одобрения начинает делать
- **Code Review** — Claude сам проверяет свой код на ошибки, как второй разработчик
- **Debugging** — системный подход к поиску багов вместо тыканья наугад
- **Planning** — разбивает большую задачу на шаги и выполняет по порядку

**Как установить Superpowers:**

**Шаг 1.** Подключи маркетплейс Superpowers. Набери в Claude Code:
```
/plugin
```
Выбери "Marketplaces" → "Add" → введи:
```
obra/superpowers-marketplace
```

**Шаг 2.** Установи плагин. Снова набери:
```
/plugin
```
Выбери "Discover" → найди **superpowers** → установи.

Готово! Теперь у тебя есть 14 новых команд. Например:
- `/superpowers:brainstorming` — обсудить идею перед реализацией
- `/superpowers:requesting-code-review` — попросить ревью кода
- `/superpowers:systematic-debugging` — найти баг по системе

**Где искать другие плагины:**

Есть сайт **claudemarketplaces.com** — агрегатор с 800+ плагинами. Там можно найти плагины под любую задачу. Но главный совет от сообщества: **5 хороших плагинов лучше 20 случайных**. Каждый плагин занимает "внимание" Claude, и если их слишком много — он начинает путаться.

### Свои скиллы: научи Claude делать то, что нужно именно тебе

Можно создавать **собственные команды**. Например:
- `/fact-check` — Claude проверяет свои же утверждения через интернет (ловит галлюцинации)
- `/save-context` — сохраняет всё важное из разговора в файлы, чтобы в следующей сессии не начинать с нуля
- `/web-research [тема]` — исследование темы с несколькими источниками

Скилл — это просто текстовый файл с инструкциями. Никакого программирования не нужно — пишешь обычным языком что Claude должен делать, и он делает.

**Как создать свой скилл:**
1. Создай папку: `~/.claude/skills/название-скилла/`
2. Внутри создай файл `SKILL.md`
3. В начале файла — шапка с настройками (имя, описание, какие инструменты разрешены)
4. Дальше — инструкции для Claude на обычном языке

Можешь даже попросить Claude: "создай мне скилл который делает X" — и он сам напишет этот файл за тебя.
